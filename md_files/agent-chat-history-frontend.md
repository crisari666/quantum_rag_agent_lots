# Agent chat — conversation history (frontend)

This document describes how **persisted** agent chat messages work: the `conversationId` thread, **`POST /rag/agent-chat/ask`** persistence, and **`GET .../messages`** (last **10** messages).

For the full ask contract (validation, `sources`, smoke test), see [`agent-chat-frontend.md`](./agent-chat-frontend.md).

## Concepts

- Each **conversation** is identified by a **UUID v4** (`conversationId`).
- The server stores **pairs** of messages after each successful ask: one **user** row (the `question`) and one **assistant** row (the `output`, plus **sources** when RAG was used).
- **Listing** for the UI uses the messages endpoint, which returns at most the **10 most recent** stored messages, **oldest first**.
- **Agent context** when you continue a thread: if the client sends `conversationId` on ask, the server loads up to **50** prior stored messages from MongoDB and **ignores** the request body `chatHistory`. If `conversationId` is omitted, the server uses `chatHistory` from the body only (nothing is read from the DB for context).

## `POST /rag/agent-chat/ask` — thread id and persistence

### Extra request field

| Field             | Type   | Required | Description |
|-------------------|--------|----------|-------------|
| `conversationId`  | string | no       | UUID v4. Omit on the **first** message; send the value returned by the API on every **subsequent** message in the same chat. |

### Response fields (addition)

| Field              | Type   | Description |
|--------------------|--------|---------------|
| `conversationId`   | string | UUID v4 for this thread. Always present. On first ask (when the client omitted `conversationId`), this is a **new** id generated by the server. |

After a successful `200`, the user question and assistant answer for that call are **appended** to this conversation in the database.

### First message without `conversationId`

- You may still send `chatHistory` for LLM context; that history is **not** written to the database—only the current `question` and `output` are stored.
- The response includes a new `conversationId`; persist it client-side for follow-up asks and for the messages endpoint.

### Follow-up with `conversationId`

- Send the same `conversationId` and **do not** rely on `chatHistory` for server-side continuity (it is ignored when `conversationId` is present).
- The server loads up to **50** stored messages as context, then runs the agent with the new `question`.

## `GET /rag/agent-chat/conversations/:conversationId/messages`

Returns up to the **last 10** persisted messages for the conversation, in **chronological** order (oldest → newest).

### Path parameter

| Parameter          | Type   | Description |
|--------------------|--------|-------------|
| `conversationId`   | string | UUID v4 (same as returned by `POST .../ask`). Invalid UUID → **400**. |

### Success response

**Status:** `200 OK`

```json
{
  "messages": [
    {
      "role": "user",
      "content": "Which projects have a pool?",
      "createdAt": "2026-03-29T12:00:00.000Z"
    },
    {
      "role": "assistant",
      "content": "…",
      "createdAt": "2026-03-29T12:00:05.000Z",
      "sources": ["https://example.com/brochure.pdf"]
    }
  ]
}
```

| Field        | Applies to   | Description |
|--------------|--------------|-------------|
| `role`       | all          | `"user"` or `"assistant"`. |
| `content`    | all          | Message text. |
| `createdAt`  | all          | ISO 8601 timestamp (server time). |
| `sources`    | assistant    | RAG **source** strings for that reply; **empty array** if none (structured-only answer). Omitted on **user** messages. |

If the conversation has **no** rows yet (e.g. typo id or never saved), `messages` is an **empty array** (`[]`).

### Error responses

| Status | When |
|--------|------|
| `400`  | `conversationId` is not a valid UUID v4. |

## Suggested client flow

1. **First ask:** `POST /ask` with `{ question }` (optional `chatHistory`). Store `conversationId` from the response.
2. **Show sidebar / history:** `GET /conversations/{conversationId}/messages` to render the latest **10** turns (adjust UI if you need more than 10 stored rows visible—only 10 are returned).
3. **Next ask:** `POST /ask` with `{ conversationId, question }` (no need to send `chatHistory` for server continuity).

---

## Limits (reference)

| Limit | Value | Where |
|-------|-------|--------|
| Messages returned by GET | **10** | `AgentChatHistoryService` |
| Messages loaded for agent context | **50** | Same service, when `conversationId` is sent on ask |
| `chatHistory` items on ask (when no `conversationId`) | **50** | `AskAgentDto` validation |

## TypeScript types (reference)

```typescript
export interface AgentChatHistoryUserMessage {
  role: 'user';
  content: string;
  createdAt: string;
}

export interface AgentChatHistoryAssistantMessage {
  role: 'assistant';
  content: string;
  createdAt: string;
  sources: string[];
}

export type AgentChatHistoryMessage =
  | AgentChatHistoryUserMessage
  | AgentChatHistoryAssistantMessage;

export interface AgentChatMessagesResponse {
  messages: AgentChatHistoryMessage[];
}

export interface AskAgentResponseWithThread {
  output: string;
  sources: string[];
  conversationId: string;
}
```
